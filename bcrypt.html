Bcrypt

Bcrypt is a modern password hashing function designed to be slow, adaptive, and resistant to brute-force and GPU attacks.

What is Bcrypt?

Bcrypt is based on the Blowfish cipher and introduced in 1999. It is specifically designed for hashing passwords — not for file integrity, signatures, or general hashing. It includes a built-in salt and a configurable cost factor, making attacks exponentially harder as hardware improves.

Key Features

  • Adaptive cost factor (work factor)
  • Built-in 128-bit salt
  • Slow and intentionally CPU-heavy to resist brute-force attacks
  • Resistant to GPU/ASIC cracking due to memory and computation patterns

Bcrypt Hash Structure

A bcrypt hash typically looks like this:

$2b$12$EIXChE5pUx0DPKySx9/TeO3Fq2IYyqSiiMHDL/95B2S/bR5CV2NnO
  • $2b$ → Version
  • 12 → Cost factor (2^12 rounds)
  • Salt → 128 bits
  • Hash → 184 bits

Why Bcrypt Is Secure

Bcrypt slows down attackers by design. Increasing the cost factor makes each hash calculation exponentially more expensive. For example:

Cost 10 → 1,024 iterations
Cost 12 → 4,096 iterations
Cost 14 → 16,384 iterations

While MD5 or SHA-256 can be computed billions of times per second on GPUs, bcrypt may only manage a few hundred hashes per second — even on strong hardware.

Hashcat & Bcrypt

Bcrypt is **very slow** to crack in Hashcat. GPU optimizations provide small benefits compared to other algorithms. The main problem is, that it is also very easy to crack, it just takes time.
