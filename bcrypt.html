Bcrypt

Bcrypt is a password hashing function designed to store passwords securely. Unlike general-purpose hash functions such as SHA or MD5, Bcrypt is intentionally slow and can be configured to be more computationally expensive over time, which protects against brute-force attacks.

How Bcrypt Works

Bcrypt is based on the Blowfish cipher. It applies multiple rounds of key expansion and encryption to produce a hash. Each password hash includes a salt and a cost factor that determines the number of rounds and thus the computation time.

Key Features

  • Salting: Each password has a unique salt, preventing rainbow table attacks.
  • Configurable cost: The number of hashing rounds can be increased as hardware improves.
  • Resistance to brute-force: Slower hashing makes large-scale attacks more difficult.

Usage

Bcrypt is widely used in modern web applications for password storage. Common libraries exist for PHP, Python, JavaScript, and many other languages. A typical hashed password looks like this:

$2b$12$N9qo8uLOickgx2ZMRZo5i.uJxY5a5IXrF2Gx0zFZ1Ijq5g2d5Vf1K

Security Considerations

  • Always use a unique salt for each password.
  • Choose a sufficiently high cost factor based on your server capabilities.
  • Regularly update the cost factor as hardware improves.

Conclusion

Bcrypt is currently considered a strong choice for password hashing due to its configurable cost, salting, and resistance to brute-force attacks. It is a practical solution for secure user authentication.
