Modern Hash Functions

Modern cryptographic hash functions are designed to be secure against known attack techniques and efficient on modern hardware. They aim to provide strong collision resistance, preimage resistance, and protection against length-extension attacks.

SHA-2 Family

The SHA-2 family includes SHA-224, SHA-256, SHA-384, and SHA-512. These functions are widely used today in TLS, SSL, digital signatures, and blockchain systems such as Bitcoin. They are currently considered secure with no practical attacks known.

SHA-3 (Keccak)

SHA-3 was selected through an open competition organized by NIST. Based on the Keccak sponge construction, it differs significantly from SHA-1 and SHA-2. It offers strong security margins, resistance to length-extension attacks, and additional variants such as SHAKE128 and SHAKE256.

BLAKE & BLAKE2

The original BLAKE hash function was a finalist in the SHA-3 competition. Its successor, BLAKE2, provides even better performance and flexibility while maintaining the same strong security guarantees. BLAKE2 is faster than MD5, SHA-1, and SHA-2 on most software platforms.

Other Modern Hash Functions

  • Whirlpool: A 512-bit hash based on a modified AES-like design.
  • Skein: Another SHA-3 finalist with a tweakable design for added flexibility.
  • RIPEMD-160: Still used in some applications such as Bitcoin addresses, though less common today.

Key Security Properties

  • Collision resistance: Hard to find two different inputs with the same hash.
  • Preimage resistance: Hard to derive the original message from its hash.
  • Second preimage resistance: Hard to find a different message with the same hash as a given message.
  • Resistance to length-extension attacks: Many modern hashes (e.g., SHA-3, BLAKE2) avoid this issue.
